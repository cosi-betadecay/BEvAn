## Halt summary (after iter 12) — soft halt at a justified optimum

**Best: iter 9, MCC=0.8773** (committed/pushed). Total V2 gain **+0.0110** over the post-Phase-0 baseline (0.8663 → 0.8773); recall improved 0.9370 → 0.9552, precision 0.8248 → 0.8276. **This is the first real improvement across V1+V2** — V1 halted at baseline after 10 straight reverts.

**Not a hard stop condition** (only 3 consecutive no-best, well short of 10). This is a *deliberate* soft halt: all three reachable verbs are exhausted and the last 3 iters are confirmatory micro-variations, so continuing would be the "tiny-steps / spinning" anti-pattern. Writing the situation here per the loop's halt-and-redirect mechanism.

**Phase 0 finding.** Both mandated metric fixes (NaN-mask in confusion matrix, full-population prior) are *no-ops on this data*: `lookup_density_values` already maps NaN→0.0 density (so NaN never reaches the ratio; `NaN excluded: 0`) and `delta_E` is finite for every event (so isfinite-count == numel). V2 baseline ≡ V1 baseline. **RL_V2's "metric noise" premise does not manifest; V1's reverts were real.** Fixes kept as defensive correctness.

**What worked — the key insight.** Of the three features, only `arm` is order-sensitive (`delta_E`/`angle` are order-invariant and already globally optimal over the enumerated subset pool, so they are unreachable by augmentation, iters 2-4). All V2 gains come from controlling the orderings fed to `arm`:
- **iter 2** (+0.0049): full (first,second) ordering enumeration for size≤4 subsets → `arm`'s argmin reaches the physically-consistent ordering the arbitrary ascending enumeration hid. Recovered 7 signal events.
- **iter 7+9** (+0.0061): **CKD-consistency ordering FILTER** — keep every subset but feed `arm` only kinematically-consistent orderings (mean sq (cosφ_geo−cosφ_kin) ≤ 0.05). The breakthrough: *filter orderings, not subsets*. This sidesteps the near-zero-sum that doomed all removal (V1's 7 reverts + V2 iters 5-6: dropping a subset also raises the order-invariant `delta_E`/`angle` mins, so each FP removed costs a TP 1:1). Pruning spurious low-`arm` orderings from the train-time density fit sharpened the boundary, improving TP *and* FP together.

**What's exhausted.** (1) Augmentation of subset *membership* — iters 3,4 byte-identical (more orderings on size≥5 / larger subsets never change `arm`'s per-event min; the 7-hit cap costs no signal). (2) Subset *removal* of every kind — energy/lever-arm/n_hits (V1) and CKD-χ² (iters 5,6) — all near-zero-sum. (3) Ordering-filter refinements — threshold sweep peaks at 0.05 (iters 8,9,10); size-2 physical-E2 is a no-op (iter 11); strict unphysical rejection hurts (iter 12).

**The ceiling.** The residual 120 FP are background events that are 511-annihilation-like in *all three* features (low `delta_E`, back-to-back `angle`, now CKD-consistent `arm`). `event_processing.py` can only influence `arm` orderings; `delta_E`/`angle` are structurally fixed. **Breaking through requires a new discriminating FEATURE downstream (out of scope for this loop):** e.g. (a) a two-photon joint constraint (both annihilation photons summing to ~1022 keV, theory §7) added to `physics_factors.py`/`modeling/`; (b) below-instrument / backscatter rejection using the reconstructed first-scatter direction sign (theory §4.3), which needs the incoming unit vector that `event_processing` doesn't receive; or (c) a coherence feature requiring a *single* subset to be jointly good in ΔE+angle+arm (the current independent per-feature `min` aggregation is itself an FP source — V1 iter0 insight — and lives in the downstream aggregation, not here). **Redirect needed if the user wants to pursue these.**
