# Understanding `Activation.sim` Files in MEGAlib

This document explains the structure of a **COSIMA/MEGAlib simulation file** (`*.sim`).  
Such files are generated by [MEGAlib](https://github.com/zoglauer/megalib), based on the Geant4 toolkit, and are used for event reconstruction and imaging (e.g., with **Revan** or **Mimrec**) in the context of NASA’s **COSI project**.

---

## 1. File Header

The file begins with metadata describing the simulation setup:

```text
Type       SIM
Version    101
Geometry   /path/to/geometry/Max.geo.setup
Date       2025-08-29 13:43:47
MEGAlib    4.00.00
Seed       1756500227
BeamType   Activation
SpectralType Activation
```

* Type / Version – File type (SIM) and format version.
* Geometry – Detector geometry setup used in the simulation.
* Date / Seed – Timestamp and random seed for reproducibility.
* BeamType / SpectralType – Defines the source type and spectral shape (here: activation).

## 2. Events (SE Blocks)

```text
SE
ID 1 1
TI 0.030484860
ED 10.3314
EC 222.31
...
```

* SE – Start of Event.
* ID a b – Event number (internal identifiers).
* TI – Event time (seconds).
* ED – Deposited energy (keV).
* EC – Total energy of the particle (keV).
* NS – Number of secondaries.
* PM – (Optional) Material-related information, e.g., activation.

## 3. Interactions (`IA` Lines)
Within each event, individual interaction steps are listed with `IA`:
* `IA INIT` – Initialization of the primary particle.
* `IA DECA` – Radioactive decay producing daughter particles.
* `IA PHOT` – Photoelectric absorption.
* `IA COMP` – Compton scattering.
* `IA ANNI` – Annihilation (e.g., e⁺ + e⁻ → gammas).
* `IA BREM` – Bremsstrahlung.
* `IA ESCP` – Particle escapes the detector volume.

Each line encodes:
* Interaction ID and parent/child references.
* Time of occurrence.
* Position `(x, y, z)` in cm?
* Direction vectors.
* Energy (keV).
* Additional process-specific flags.

Example:
```text
IA DECA  2; 1;3;0.000000000000e+00; -2.92107; -3.59403; -3.50111; ...
```
This describes a decay event at a given position, producing secondary particles with defined directions and energies.

## 4. Hit Traces (`HTsim` Lines)
At the end of an event, Hit Traces summarize linked interaction chains:
```text
HTsim 3; 1.01750; -2.31250; 4.59804; 66.71816; 0.00000e+00; 2;3;4;5;6;7;8;9;10
```
* The first number is a trace ID
* Followed by spatial coordinates and deposited energy
* The last part lists the IDs of interactions belonging to this trace.

## Usage in MEGALIB
- `.sim` files are **not analyzed manually** but processed by MEGAlib tools:  
  - `revan` – Event reconstruction  
  - `mimrec` – Image reconstruction  
  - `simdeluxe` – Advanced simulation analysis  

- To study an event manually:  
  1. Start at `IA INIT` (primary particle creation)  
  2. Follow subsequent `IA DECA`, `IA COMP`, `IA PHOT`, etc.  
  3. Stop when `IA ESCP` indicates the particle left the geometry  
  4. Use `HTsim` for grouped tracks
