## Halt summary (after iter 10)

**Stop condition hit:** 10 consecutive iterations without a new best. **Best remains iter 0 (baseline, no filter), MCC=0.8663.** event_processing.py reverted to baseline; nothing committed/pushed (no new best).

**What was tried & the through-line:**
- **Energy thresholds on subset ΣE** (iters 1,2,4,5): all fail. ΣE-windowing is incompatible with the `arm` feature, whose kinematic angle depends on `E_2 = ΣE − E_first` (any ΣE can yield a physical, signal-like arm), and with `annihilation_angle`, which needs the ≈1022 keV two-photon subset. **ΣE-thresholding is a dead end.**
- **Geometric / kinematic per-subset removal** (iters 3,6,7): all fail. Removing a subset only ever *raises* a feature's min, and signal vs background argmin subsets overlap heavily in feature space, so every removal trades TP and FP ~evenly. Two compounding facts: (a) the Bayesian densities are **re-estimated from the training split with the filter applied**, so filters reshape the decision boundary non-monotonically (removal can *raise* FP); (b) the spurious-low-arm clamp-artifact subsets are largely size≥3 — the **same** subsets `annihilation_angle` needs (dual-use). The best removal (iter 7, b2b-exempted) still netted −0.016.
- **Event-level multiplicity reject** (iters 8,9,10): the only lever that moved FP substantially (N≥8 → FP 120→88). But MCC is very TP-sensitive (positives are rare), and the rejected high-multiplicity tail holds proportionally too many genuine β⁺ events. Closest was N≥10 (MCC 0.8639, −0.0024). No threshold beats baseline.

**Conclusion / recommendation for the user.** The baseline reconstruction appears to be at a strong local optimum for the interventions reachable from `event_processing.py` alone (which can only *remove subsets* or *reject whole events*). The fundamental blocker is the downstream `min`-over-subsets feature aggregation combined with train-time density re-fitting: removal cannot make signal *more* signal-like, only background *less*, and the two classes overlap. **The biggest available gains likely require changes outside `event_processing.py`** — e.g. (1) replacing the crude 2-hit `arm` with a real multi-scatter CKD χ² that uses physical hit *ordering* (theory §2, ranked #1) so the kinematic-consistency feature itself discriminates better; or (2) feeding the classifier a per-event multiplicity feature so it can down-weight high-mult events softly instead of a hard event reject. Both need edits to `physics_factors.py` / `modeling/` and so are out of scope for this loop. Redirect needed if the user wants to pursue them.
